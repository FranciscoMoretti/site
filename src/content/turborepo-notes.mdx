# **Running Tasks in a Monorepo**

Every monorepo has two main building blocks: **workspaces** and **tasks**. Let's imagine you have a monorepo containing **three workspaces**, each with **three tasks**:

[https://turbo.build/_next/image?url=%2F_next%2Fstatic%2Fmedia%2Fyour-monorepo-excalidraw.ccf1c6c1.png&w=3840&q=75](https://turbo.build/_next/image?url=%2F_next%2Fstatic%2Fmedia%2Fyour-monorepo-excalidraw.ccf1c6c1.png&w=3840&q=75)

Here, both `apps/web` and `apps/doc` use code from `packages/shared`. In fact, when they're built (via `build`) **they need `packages/shared` to be built *first***.

### Turbo supports virtual pipelines

`turbo` supports "virtual" pipelines so even if the pipeline does not have a matching script on a package it still uses it to decide what to run.

[https://github.com/vercel/turbo/issues/2988](https://github.com/vercel/turbo/issues/2988)
